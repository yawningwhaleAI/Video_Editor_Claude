## Step 3: Research (MANDATORY -- ALWAYS DO THIS)

**Before writing a single line of code, ALWAYS research the topic on the web.** This is not optional. Even if you think you know, verify.

### 3a: Topic Research
- WebSearch the topic/product/brand mentioned in the script
- Understand what it is, how it works, key features -- so your visuals are accurate
- If the script mentions a product (e.g., Coresignal, Replit, Vercel), research what it actually does

### 3b: Logo Research (MANDATORY for any brand/product)
- **WebSearch for the exact SVG path data** from Simple Icons (https://simpleicons.org) or the brand's GitHub repo
- Search query: `"<brand> svg path" site:simpleicons.org` or `"<brand> logo svg path data"`
- Extract the EXACT `d="..."` attribute from the SVG -- do NOT approximate or draw logos freehand
- Use `new Path2D(svgPathString)` to render on Canvas 2D
- **NEVER use PNG/image assets for logos** -- always SVG paths via Canvas 2D `Path2D` API
- **NEVER guess or approximate a logo** -- if you can't find the exact SVG path, tell you

### 3c: Design Research (pixel-perfect)
- WebSearch for exact hex colors, dimensions, typography, corner radii, spacing
- Get platform-specific measurements (iOS notification height, Slack sidebar width, etc.)
- **Every visual must be pixel-perfect** -- research actual screenshots, design specs, Apple HIG, Material Design, etc.
- If referencing any UI (macOS window, terminal, VS Code, etc.), look up the exact pixel values -- never estimate

### 3d: Google Trends Validation (MANDATORY for new content ideas)
**ALWAYS pull real Google Trends data before committing to a topic.** Use the SerpAPI Google Trends API to validate that the topic has real search interest and is trending in the right direction.

**API endpoint**: `https://serpapi.com/search.json?engine=google_trends&q=<topics>&date=today+3-m&api_key=<SERPAPI_KEY>`

- **Compare up to 5 topics** in one query (comma-separated)
- **Use `date=today+3-m`** for 3-month trend (best for content decisions)
- **Look for**: Average interest score, peak value, growth trajectory (surging vs flat vs declining)
- **Reject topics** with flat/declining trends unless they have very high baseline interest
- **Prioritize topics** with 100+ peak and clear upward trajectory
- Present the trend data to you in a table format before finalizing topics

The SerpAPI key is stored in the project memory at `memory/api-keys.md`. Always use WebFetch to call the API.

**Example query for Claude Code content:**
```
https://serpapi.com/search.json?engine=google_trends&q=claude+code+mcp,claude+code+hooks,claude+code+plugins&date=today+3-m&api_key=<KEY>
```

This step applies to content ideation and topic selection. Skip it only when you has already decided on the exact topic and just needs it built.

## Step 4: Build Everything

**MANDATORY: Use parallel subagents for EVERYTHING possible.** This is non-negotiable.

- **Research phase**: If researching 4+ topics, launch 4+ parallel agents (one per topic)
- **Build phase**: If building slides 3-8, launch 6 parallel agents (one per slide)
- **Research + build**: Launch research agent AND build agent simultaneously
- **Never sequential**: If two tasks don't depend on each other, they MUST run in parallel
- **Single message, multiple Agent calls**: Always send one message with multiple Agent tool blocks

you expects maximum parallelism at all times. Sequential work when parallel is possible = wasted time.

  const scale = Math.max(CW / wallpaper.naturalWidth, CH / wallpaper.naturalHeight);

### drawScene Function (Canvas 2D -- the core)

5-param signature for ReelExportButton:
```typescript
function drawScene(
  ctx: CanvasRenderingContext2D,
  entryProgress: number,      // 0-1 during entry
  holdProgress: number,       // 0-1 during hold
  exitProgress: number,       // 0-1 during exit
  wallpaper: HTMLImageElement | null
) {
  ctx.clearRect(0, 0, CW, CH);

  // 1. Wallpaper + dark overlay (Canvas 2D)
  // ... (see wallpaper pattern above)

  // 2. Spring entry transform (Canvas 2D transforms)
  const sp = springInterpolate(entryProgress, { stiffness: 280, damping: 22, mass: 1 });
  const alpha = easings.easeOutQuad(Math.min(1, entryProgress * 2.5));
  const entryScale = 0.85 + 0.15 * Math.min(1.02, sp);

  ctx.save();
  ctx.globalAlpha = alpha;
  ctx.translate(CW / 2, CH / 2);
  ctx.scale(entryScale, entryScale);
  ctx.translate(-CW / 2, -CH / 2);

  // 3. ALL CONTENT drawn with Canvas 2D API:
  //    ctx.fillRect(), ctx.fillText(), ctx.drawImage(),
  //    roundedRect(), ctx.beginPath(), ctx.arc(),
  //    new Path2D(svgString) for logos,
  //    ctx.createLinearGradient() for gradients

  // 4. Sequenced reveals during hold phase:
  //    if (holdProgress > 0.3) { ... draw element 2 ... }
  //    if (holdProgress > 0.6) { ... draw element 3 ... }

  ctx.restore();
}
```

### Easing Functions Available
```typescript
easings.linear          // t
easings.easeInQuad      // t^2
easings.easeOutQuad     // t(2-t) -- most common for fade-in
easings.easeInOutQuad   // smooth both
easings.easeInCubic     // t^3
easings.easeOutCubic    // (t-1)^3+1 -- MOST USED for deceleration

```

### Animation Sequencing Patterns

**Time-based sequencing (preferred for complex animations)**:
```typescript
const tProg = (t: number, s: number, d: number) => Math.max(0, Math.min(1, (t - s) / d));

// Title at t=0s, duration 0.6s
const titleP = springInterpolate(tProg(currentTime, 0, 0.6), SPRING);

// Subtitle at t=0.5s, duration 0.5s
const subP = springInterpolate(tProg(currentTime, 0.5, 0.5), SPRING);
```

**holdProgress-based sequencing (for simpler animations)**:
```typescript
// Element 1: holdProgress 0.0-0.3
if (holdProgress > 0) {
  const p = easings.easeOutCubic(Math.min(1, holdProgress / 0.3));
  ctx.globalAlpha = p;
  // draw element 1...
}

// Element 2: holdProgress 0.3-0.6
if (holdProgress > 0.3) {
  const p = easings.easeOutCubic(Math.min(1, (holdProgress - 0.3) / 0.3));
  ctx.globalAlpha = p;
  // draw element 2...
}
```

**Staggered items**:
```typescript
items.forEach((item, i) => {
  const delay = 0.05 * i;
  const p = Math.max(0, Math.min(1, (holdProgress - 0.2 - delay) / 0.1));
  const alpha = easings.easeOutQuad(p);
  ctx.globalAlpha = alpha;
  // draw item...
});
```

export default function PartN() {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const loopRef = useRef<{ stop: () => void; restart: () => void } | null>(null);
  const wallpaperRef = useRef<HTMLImageElement | null>(null);

  useEffect(() => {
    const canvas = canvasRef.current;
    if (!canvas) return;

    // DPR scaling for crisp Retina preview (MANDATORY)
    const dpr = window.devicePixelRatio || 1;
    canvas.width = CW * dpr;
    canvas.height = CH * dpr;

    const ctx = canvas.getContext('2d');
    if (!ctx) return;
    ctx.scale(dpr, dpr);

    // Load wallpaper
    const img = new Image();
    img.crossOrigin = 'anonymous';  // REQUIRED for external URLs
    img.src = WALLPAPER_URL;
    img.onload = () => { wallpaperRef.current = img; };
    wallpaperRef.current = img;

    // Animation loop (Canvas 2D frame-by-frame)
    const loop = createPlaybackLoop(
      ANIM_CONFIG,
      (frame) => {
        ctx.setTransform(dpr, 0, 0, dpr, 0, 0); // Reset DPR transform each frame
        drawScene(ctx, frame.entryProgress, frame.holdProgress, frame.exitProgress, wallpaperRef.current);
      },
      () => {} // No loop, no fade-out (reel rules)
    );
    loopRef.current = loop;
    return () => loop.stop();
  }, []);

  return (
    <div style={{
      minHeight: '100vh', background: '#000',
      display: 'flex', flexDirection: 'column', alignItems: 'center', justifyContent: 'center',

        <span style={{ color: 'rgba(255,255,255,0.15)', fontSize: 12 }}>|</span>
        <span style={{ color: 'rgba(255,255,255,0.3)', fontFamily: '-apple-system', fontSize: 12 }}>

Bitrate: 28 Mbps at 1080p (high quality), scaled proportionally for other resolutions.

export const CH = 1440;
export const TOTAL_SLIDES = N;

// Typography (for ctx.font)

export const MONO = 'Menlo, Monaco, Consolas, monospace';

// Colors (for ctx.fillStyle)
export const BG = '#F4F3EE';
export const ACCENT = '#C15F3C';
export const TEXT_PRIMARY = '#333333';
export const TEXT_SECONDARY = '#666666';

// Canvas 2D drawing helpers
export function drawSlideDots(ctx: CanvasRenderingContext2D, current: number, total: number, y: number) {
  const dotR = 4;
  const gap = 14;
  const totalW = (total - 1) * gap;
  const startX = (CW - totalW) / 2;
  for (let i = 0; i < total; i++) {
    ctx.beginPath();
    ctx.arc(startX + i * gap, y, dotR, 0, Math.PI * 2);
    ctx.fillStyle = i === current - 1 ? ACCENT : 'rgba(0,0,0,0.15)';
    ctx.fill();
  }
}

'use client';

import { useCallback, useEffect, useRef, useState } from 'react';
import { downloadBlob } from '@/lib/export/videoExporter';
import { CW, CH, SF, BG, ACCENT, TOTAL_SLIDES, drawSlideDots } from '../_shared';

const SLIDE_NUM = 1;
const FILENAME = '<series>-carousel-01.png';

// Canvas 2D render function -- ALL content drawn here
function renderSlide(ctx: CanvasRenderingContext2D) {
  ctx.clearRect(0, 0, CW, CH);

  // Background (Canvas 2D)
  ctx.fillStyle = BG;
  ctx.fillRect(0, 0, CW, CH);

  // === ALL CONTENT DRAWN WITH CANVAS 2D API ===
  // ctx.fillText(), ctx.fillRect(), roundedRect(),
  // ctx.drawImage(), new Path2D(), ctx.arc(), etc.

  // Slide dots at bottom (Canvas 2D)
  drawSlideDots(ctx, SLIDE_NUM, TOTAL_SLIDES, CH - 60);

export default function Slide1() {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const [exporting, setExporting] = useState(false);

  useEffect(() => {
    const canvas = canvasRef.current;
    if (!canvas) return;
    const ctx = canvas.getContext('2d');
    if (!ctx) return;
    renderSlide(ctx);  // One-time Canvas 2D render
  }, []);

  // PNG Export: Canvas 2D -> OffscreenCanvas at 4x -> PNG blob -> download
  const handleExport = useCallback(async () => {
    if (exporting) return;

    try {
      const dpr = 4;  // 4x scale -> 4320x5760 final
      const offscreen = new OffscreenCanvas(CW * dpr, CH * dpr);
      const offCtx = offscreen.getContext('2d') as unknown as CanvasRenderingContext2D;

      renderSlide(offCtx);     // Re-render at high res
      const blob = await offscreen.convertToBlob({ type: 'image/png' });
      await downloadBlob(blob, FILENAME);
    } catch (err) {
      console.error('[PNG export]', err);
    } finally {
      setExporting(false);
    }
  }, [exporting]);

  return (
    <div style={{
      minHeight: '100vh', background: 'radial-gradient(ellipse at 50% 30%, #F5F3EE 0%, #E8E4DB 100%)',

          SERIES NAME -- SLIDE {SLIDE_NUM} OF {TOTAL_SLIDES}

      </div>

        style={{ width: '100%', maxWidth: 500, height: 'auto', borderRadius: 16, boxShadow: '0 2px 4px rgba(0,0,0,0.04), 0 12px 40px rgba(0,0,0,0.06)' }}

        <button onClick={handleExport} disabled={exporting}

            width: '100%', height: 52, borderRadius: 12, border: 'none',
            background: '#FF6B35', color: '#FFF', fontSize: 15, fontWeight: 700,
            cursor: exporting ? 'default' : 'pointer', fontFamily: SF,

        &nbsp; &middot; &nbsp; 1080 x 1440
      </p>
    </div>
  );
}

## CRITICAL RULES

### Pixel-Perfect & Highest Resolution (MANDATORY)
0. **EVERYTHING must be pixel-perfect.** Research exact dimensions, colors, fonts, spacing, corner radii before drawing anything. Never estimate. Never approximate. If you're referencing a UI (macOS, iOS, VS Code, terminal), look up the exact pixel values.
0b. **ALWAYS export at 4x resolution.** Carousel PNG: `dpr = 4` (4320x5760). Landscape video: `exportScale={4}` (7680x4320). Portrait reel: native 1080x1920.
0c. **ALWAYS research the topic on the web BEFORE writing code.** Understand what the script is about. Research brand logos, colors, and SVG paths. This is not optional.
0d. **ALWAYS use DPR scaling on preview canvases (MANDATORY).** Every preview canvas must scale by `window.devicePixelRatio` for crisp Retina rendering. Pattern:
```typescript
const dpr = window.devicePixelRatio || 1;
canvas.width = CW * dpr;
canvas.height = CH * dpr;
const ctx = canvas.getContext('2d');
ctx.scale(dpr, dpr);
// In animation loop, reset each frame:
ctx.setTransform(dpr, 0, 0, dpr, 0, 0);
```
CSS size stays the same (controlled by style). Only the canvas buffer is scaled. This does NOT affect export resolution.

### Canvas 2D Engine (MANDATORY)
1. **ALL rendering uses Canvas 2D.** Every pixel is drawn via `CanvasRenderingContext2D`. No HTML/CSS mockups. No DOM-based visuals. No WebGL/Three.js.
2. **Import from `@/lib/engine/`** -- the animation engine already exists:
   - `createPlaybackLoop` from `@/lib/engine/animation-loop`
   - `springInterpolate` from `@/lib/engine/spring`
   - `easings` from `@/lib/engine/easing`
   - `roundedRect` from `@/lib/engine/drawing-utils`
   - `AnimationConfig` type from `@/lib/templates/types`
3. **NEVER recreate the engine.** Never write your own spring physics, easing functions, or animation loop. Import from the existing engine.
4. **NEVER recreate export buttons.** Use `ReelExportButton` (portrait) or `LandscapeExportButton` (landscape) from `@/components/reel-export/`.
5. **Always `ctx.save()` / `ctx.restore()`** around Canvas 2D state changes (transforms, alpha, clip, font).
6. **SVG logos via Path2D**: `const path = new Path2D(svgPathString); ctx.fill(path);` -- never use PNG images for logos.
